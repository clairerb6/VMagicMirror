# URP移行タスク

## 現状整理

- Unity プロジェクト全体としては URP で起動できており、アバターのロードと通常動作は確認済み。
- アバターには URP の MToon が割り当てられているように見え、シェードも効いていることを確認済み。
- アバターのアウトライン表示、ポスプロ適用、透過出力、透過出力時の切り抜き、SSAO、アンチエイリアス設定は確認済み。
- アクセサリーのマテリアル割り当て、とくに MToon 系の割り当ては確認済み。
- サブキャラのマテリアル割り当ても確認済み。
- キーボード / マウスなどのデバイスオブジェクト表示、スクリーンショット機能も確認済み。
- 一方で、固定影の板ポリ描画と品質ごとの URP Asset 切り替えは未解決または要確認。

## 現時点の未解決タスク

### P0: 品質設定ごとの URP Asset 定義を固める

- `ProjectSettings/QualitySettings.asset` における品質ごとの `customRenderPipeline` の実運用方針を決める。
- 単一の URP Asset を全品質で使うか、品質別に URP Asset を分けるかを決める。
- スクリプトで差し替える場合も含め、実際の切り替え経路を明示する。

### P1: 固定影 (ShadowDrawer) を確認・復旧する

- 陰を板ポリに描く機能が URP で正しく動くか確認する。
- `MainLight` ベースの通常 shadow とは分離し、`ShadowLight` 視点の専用 shadow map を `ShadowBoard` 用に生成する方針で進める。
- 受け側は「投影マスクを貼る」のではなく、`ShadowBoard` の各ピクセルを `ShadowLight` 空間へ変換して shadow map と深度比較する方式を採る。
- caster を「アバター限定」で決め打ちする方針は採らず、距離ベースの除外ルールで shadow map に描く対象を決める。
  - 近傍の renderer はアバター、アクセサリー、必要ならサブキャラも自然に含める。
  - 遠景オブジェクトは `maxCasterDistance` 相当のルールで除外する。
  - layer は補助的に使えてもよいが、主たる選別ルールは距離と除外条件に寄せる。

## 完了済みとして扱ってよい項目

- URP でクラッシュせず起動し、アバターがロードされる。
- アバターへの URP MToon 適用。
- MToon のシェード確認。
- アバターのアウトライン表示。
- ポスプロの適用。
- 透過出力。
- 透過出力時の切り抜き機能。
- Screen Space Ambient Occlusion の適用。
- アンチエイリアス設定の適用。
- アクセサリーのマテリアル割り当て。
- サブキャラのマテリアル割り当て。
- キーボード / マウス等のオブジェクト表示。
- スクリーンショット機能。
- Spout 出力。
- `uDesktopDuplication` を用いたデスクトップ色調補正。

## 優先順位

1. 品質設定ごとの URP Asset 運用を固める
2. 固定影 (ShadowDrawer) の動作確認と復旧
3. 残件の回帰確認と品質差分の調整
