# URP移行タスク

## 現状整理

- Unity プロジェクト全体としては URP で起動できており、アバターのロードと通常動作は確認済み。
- アバターには URP の MToon が割り当てられているように見え、シェードも効いていることを確認済み。
- アバターのアウトライン表示、ポスプロ適用、透過出力、透過出力時の切り抜き、SSAO、アンチエイリアス設定は確認済み。
- アクセサリーのマテリアル割り当て、とくに MToon 系の割り当ては確認済み。
- サブキャラのマテリアル割り当ても確認済み。
- キーボード / マウスなどのデバイスオブジェクト表示、スクリーンショット機能も確認済み。
- 一方で、固定影の板ポリ描画、Desktop Light、Spout 使用時の描画停止、品質ごとの URP Asset 切り替えは未解決または要確認。

## 現時点の未解決タスク

### P0: 品質設定ごとの URP Asset 定義を固める

- `ProjectSettings/QualitySettings.asset` における品質ごとの `customRenderPipeline` の実運用方針を決める。
- 単一の URP Asset を全品質で使うか、品質別に URP Asset を分けるかを決める。
- スクリプトで差し替える場合も含め、実際の切り替え経路を明示する。

### P1: 固定影 (ShadowDrawer) を確認・復旧する

- 陰を板ポリに描く機能が URP で正しく動くか確認する。
- `ShadowDrawer.shader` と `UrpShadowDrawer.shader` のどちらが実際に使われるべきかを整理する。
- prefab / material / script の参照経路を洗い出し、URP 実行時に正しいシェーダを使うよう統一する。

### P2: Desktop Light を安全に戻すか、止めたままにするか決める

- `uDesktopDuplication` を用いたデスクトップ色調補正は、動作させるとクラッシュする挙動が観察されている。
- 当面停止するなら停止理由と影響範囲を明記する。
- 復旧するなら、`DesktopLightEstimator` の入力テクスチャ取得、フォーマット、更新タイミングを中心に切り分ける。

### P3: Spout 使用時の描画停止を調査する

- Spout 出力自体は更新されるが、使用中にウィンドウ上の描画が停止する問題が残っている。
- main camera 側と Spout 側でどの描画経路が分岐しているかを確認する。
- post processing やカメラ設定、RenderTexture 更新順の影響を切り分ける。

## 完了済みとして扱ってよい項目

- URP でクラッシュせず起動し、アバターがロードされる。
- アバターへの URP MToon 適用。
- MToon のシェード確認。
- アバターのアウトライン表示。
- ポスプロの適用。
- 透過出力。
- 透過出力時の切り抜き機能。
- Screen Space Ambient Occlusion の適用。
- アンチエイリアス設定の適用。
- アクセサリーのマテリアル割り当て。
- サブキャラのマテリアル割り当て。
- キーボード / マウス等のオブジェクト表示。
- スクリーンショット機能。

## 優先順位

1. 品質設定ごとの URP Asset 運用を固める
2. 固定影 (ShadowDrawer) の動作確認と復旧
3. Desktop Light の扱いを決める
4. Spout 使用時の描画停止を解消する

## 補足メモ

- 以前は未移植扱いだったポスプロ、透過、AA、SSAO、アクセサリー / サブキャラのマテリアル経路は、現時点では完了済みとして扱う。
- 今後は「URP に未対応な大枠の移植」よりも、「残件の個別不具合と運用整理」が主な作業になる。
